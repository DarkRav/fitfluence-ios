# Product Audit — Fitfluence iOS

## 1. Core Value
- Главный сценарий: атлет открывает тренировку и проходит упражнения с сохранением прогресса.
- До первой тренировки сейчас: вход -> onboarding (иногда выбор роли) -> каталог -> программа -> тренировки -> тренировка.
- Фрикция: слишком длинный путь до первого полезного действия, нет явной стартовой точки после логина.
- Проблема точки входа: пользователь попадает в табы без понятного «главного экрана».

## 2. Лишняя сложность
- Мобильное приложение перегружено веткой influencer onboarding, хотя основной loop — athlete workout.
- Есть таб «Мои тренировки» без реального контента (заглушка), создаёт ложное ожидание.
- Ветка выбора роли в onboarding увеличивает когнитивную нагрузку для athlete-first сценария.

## 3. Flow после логина
- Пользователь не всегда понимает, что делать первым делом.
- Нет отдельного Home с рекомендацией действия.
- Ощущение завершённости нарушено из-за тупикового таба и второстепенных экранов в основном потоке.

## 4. Workout Flow
- Быстрый старт тренировки не укладывается в <=3 клика из корня.
- Продолжение прерванной тренировки технически поддержано, но не вынесено в главный входной сценарий.
- Риск потери данных низкий (autosave), но обнаружение незавершённой тренировки скрыто в плеере, а не на Home.

## 5. Onboarding
- Полей больше, чем нужно для старта athlete.
- Цель формы понятна, но есть лишние ветки и сообщения про неактуальные роли.
- Нужно упростить до минимального athlete профиля.

## Решение (принято в этом шаге)
1. Сделать приложение athlete-first: убрать influencer ветку из mobile flow.
2. Добавить Home как главный экран с приоритетом «Продолжить тренировку».
3. Упростить onboarding до минимального набора полей.
4. Упростить навигацию: убрать тупиковый таб, сохранить только осмысленные разделы.
