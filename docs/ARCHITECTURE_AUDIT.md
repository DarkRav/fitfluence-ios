# Architecture Audit — Fitfluence iOS

## Найденные проблемы

## 1. Дублирование обработки ошибок
- В фичах и onboarding встречались отдельные маппинги APIError -> UI-текст.
- Риск расхождения поведения и копирайта между экранами.

## 2. Избыточные ветки состояния
- Root включает influencer-ориентированные переходы в основном athlete приложении.
- Лишняя ветка влияет на bootstrap/onboarding и усложняет state machine.

## 3. Навигационная избыточность
- Таб со заглушкой не имеет доменной ценности, но создаёт дополнительную ветку состояния.

## 4. Прогресс тренировки
- Autosave есть, но точка восстановления не интегрирована в верхнеуровневый флоу.
- Нет единого «resume» orchestration в Root/Home.

## 5. Logout / cache
- Очистка namespace кеша реализована корректно.
- Нужно гарантировать, что в athlete-first навигации не остаются orphan-состояния после logout.

## 6. Token/401 flow
- refresh + retry реализованы и покрыты тестами.
- Архитектурно важно не размножать логику поверх API слоя.

## План исправлений (выполняется)
1. Athlete-first root state и onboarding.
2. HomeFeature как оркестратор resume-потока.
3. Упрощение таб-навигации и удаление тупиков.
4. Централизация user-facing ошибок для всех пользовательских экранов.
5. Тесты на resume/restore + smoke QA (format/build/tests).
